### 🔵 Legal Eagle — Offline RAG for Indian Law

<img src="https://img.shields.io/badge/Domain-Legal%20AI-1565C0?style=flat-square"/> <img src="https://img.shields.io/badge/Stack-LangChain%20%2B%20ChromaDB%20%2B%20Ollama-00B4D8?style=flat-square"/> <img src="https://img.shields.io/badge/Pages%20Indexed-620+-22C55E?style=flat-square"/> <img src="https://img.shields.io/badge/API%20Cost-$0-F59E0B?style=flat-square"/>

**The problem:** Legal professionals need accurate answers over large, complex document sets — but OpenAI API costs at scale are prohibitive, and data privacy means no cloud uploads.

**What I built:** A fully offline RAG pipeline over 620+ pages of Indian legal documents. ChromaDB for vector storage, Ollama for local inference — zero external calls, zero cost at query time.

**Results:**
- ⚡ **3× faster** document ingestion via parallelized processing + batched embedding
- 🎯 **MMR retrieval** eliminated duplicate context, improved answer relevance on multi-document queries
- 🔒 **100% offline** — data never leaves the machine

### 🟣 Multi-Agent Research Commander

<img src="https://img.shields.io/badge/Domain-Agentic%20AI-7B2FBE?style=flat-square"/> <img src="https://img.shields.io/badge/Stack-LangGraph%20%2B%20FastAPI-9C27B0?style=flat-square"/> <img src="https://img.shields.io/badge/Agents-4%20Specialized-E91E63?style=flat-square"/>

### 🟢 Predictive Maintenance — ML on Real Machinery

<img src="https://img.shields.io/badge/Domain-Industrial%20ML-1B5E20?style=flat-square"/> <img src="https://img.shields.io/badge/Stack-Scikit--learn%20%2B%20FFT%20Analysis-43A047?style=flat-square"/> <img src="https://img.shields.io/badge/R²%20Score-0.94-22C55E?style=for-the-badge"/>

**The problem:** Unplanned machinery failure costs manufacturers millions. Predicting Remaining Useful Life (RUL) from vibration data — with physical test validation — is hard.

**What I built:** End-to-end ML pipeline. Raw vibration signals → FFT → statistical feature extraction → Random Forest → RUL prediction. Validated on a real VFD-controlled test rig across multiple fault scenarios.

**Results:**
- 📈 **R² = 0.94** on unseen test data — not just training performance
- 🔬 **Multi-fault validated** — tested across operating conditions, not just ideal cases
- 📊 Real physical hardware — not a Kaggle dataset

[![View Project](https://img.shields.io/badge/→%20View%20on%20GitHub-0A0E1A?style=for-the-badge&logo=github&logoColor=white)](https://github.com/AnvitDevadiga)
